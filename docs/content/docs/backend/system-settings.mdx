# 系统配置数据结构

系统配置保存在 `settings` 表中，目前只使用两行：

| key | 说明 |
| --- | --- |
| `public` | 公开配置，前端可以读取 |
| `private` | 私有配置，只给后端和管理员使用 |

## public.value

```json
{
  "modelChannel": {
    "models": [
      {
        "model": "gpt-image-2-0.5k",
        "capability": "image",
        "enabled": true,
        "giftEligible": true,
        "defaultCredits": 5
      },
      {
        "model": "gpt-image-2-1k",
        "capability": "image",
        "enabled": true,
        "giftEligible": false,
        "defaultCredits": 10
      }
    ],
    "defaultModel": "gpt-image-2-1k",
    "defaultImageModel": "gpt-image-2-1k",
    "defaultTextModel": "",
    "systemPrompt": "",
    "allowCustomChannel": false
  },
  "auth": {
    "allowRegister": true,
    "linuxDo": {
      "enabled": false
    }
  }
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `modelChannel` | object | 模型渠道公开配置组 |
| `auth` | object | 认证相关公开配置 |

`modelChannel` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `models` | object[] | 前端可展示的公开模型规格，`model` 同时作为展示文案和请求模型名 |
| `defaultModel` | string | 默认模型，从启用且可路由的 `models` 中选择 |
| `defaultImageModel` | string | 默认图片模型，从启用且 `capability=image` 的模型中选择 |
| `defaultVideoModel` | string | 默认视频模型，从启用且 `capability=video` 的模型中选择 |
| `defaultTextModel` | string | 默认文本模型，从启用且 `capability=text` 的模型中选择 |
| `systemPrompt` | string | 系统提示词 |
| `allowCustomChannel` | boolean | 当前默认关闭，前端统一走后端中转接口 |

`models` 每项字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `model` | string | 公开模型名，也是前端展示文案，例如 `gpt-image-2-0.5k`、`gpt-image-2-1k` |
| `capability` | string | 能力类型：`image`、`video`、`audio`、`text` |
| `enabled` | boolean | 是否开放给前端使用；`gpt-image-2-0.5k` 可按系统供给情况动态关闭 |
| `giftEligible` | boolean | 是否允许使用系统赠送额度；当前只用于 `gpt-image-2-0.5k` |
| `defaultCredits` | number | 默认预估算力点；实际扣费优先使用命中的私有路由价格 |

## private.value

```json
{
  "channels": [
    {
      "protocol": "openai",
      "name": "中转服务商 A",
      "baseUrl": "https://api.example.com",
      "apiKey": "sk-xxx",
      "routes": [
        {
          "model": "gpt-image-2-1k",
          "upstreamModel": "gpt-image-2",
          "credits": 10,
          "weight": 1,
          "enabled": true
        }
      ],
      "enabled": true,
      "remark": ""
    }
  ],
  "promptSync": {
    "enabled": true,
    "cron": "*/5 * * * *"
  }
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `channels` | object[] | 模型渠道列表 |
| `promptSync` | object | GitHub 远程提示词定时同步配置 |

`channels` 每项字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `protocol` | string | 协议，当前为 `openai` |
| `name` | string | 渠道名称 |
| `baseUrl` | string | OpenAI 兼容接口地址 |
| `apiKey` | string | 渠道密钥 |
| `routes` | object[] | 该渠道可服务的公开模型路由 |
| `enabled` | boolean | 是否启用 |
| `remark` | string | 备注 |

`routes` 每项字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `model` | string | 公开模型名，必须匹配 `public.value.modelChannel.models[].model` |
| `upstreamModel` | string | 实际发送给上游的模型名；当前 GPT Image 2 规格统一为 `gpt-image-2` |
| `credits` | number | 命中该服务商路由时实际扣除的算力点，支持不同服务商单独定价 |
| `weight` | number | 同一公开模型命中多个可用路由时的加权随机权重 |
| `enabled` | boolean | 是否启用该路由 |

后端调用模型时，先用公开模型名筛选启用的公开规格，再从启用、配置了 `baseUrl` 和 `apiKey`、且包含对应 `routes.model` 的渠道路由中按 `weight` 选择一个。请求发往上游前会把请求体里的 `model` 改写为命中路由的 `upstreamModel`，因此前端和用户只需要感知 `gpt-image-2-0.5k`、`gpt-image-2-1k` 等公开规格。

赠送额度只会在公开模型 `giftEligible=true` 时参与扣费；通用额度不足但赠送额度足够时，只有这类模型可以继续调用。上游调用失败时，已扣除的通用额度和赠送额度会按原扣费桶返还。

`promptSync` 字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `enabled` | boolean | 是否开启定时同步，默认开启 |
| `cron` | string | Cron 表达式，默认每 5 分钟 |
